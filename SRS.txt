SOFTWARE REQUIREMENTS SPECIFICATION (SRS)
AI Wardrobe Stylist Mobile App (v1)

1. Introduction
1.1 Purpose
This document defines the requirements for a mobile application that acts as a personal fashion stylist. The system allows users to digitize their wardrobe, receive outfit recommendations, and interact with a basic stylist chat.

1.2 Scope
The application is a mobile-only product built using React Native and Expo. It provides wardrobe storage, outfit recommendations based on owned clothes, and a basic chat interface.

1.3 Definitions
- Wardrobe: The user's full collection, encompassing both clothing items and outfits
- Outfit: Combination of clothing items
- Item: Single clothing piece uploaded by user

2. Overall Description
2.1 Product Perspective
Standalone mobile app using Supabase backend for storage, database, and authentication.

2.2 User Characteristics
General consumers with no technical expertise required.

2.3 Constraints
- Mobile-only (React Native + Expo)
- Supabase for backend services
- AI components are stubbed (no real implementation in v1)

3. Functional Requirements

3.1 User Authentication
- Users can sign up
- Users can log in
- Users can log out

3.2 Onboarding
- Users input preferences:
  - favorite colors
  - disliked colors
  - preferred styles
  - sizes

3.3 Clothing Item Management
- Users can take photos
- Users can upload images
- System shows detected tags
- Users can edit tags
- Users can save items to closet

3.4 Wardrobe Management
- Users can view clothing items in a grid under the Wardrobe tab
- Users can view outfits in a list under the Wardrobe tab
- Users can filter clothing items by:
  - category
  - color
  - style

3.5 Item Detail
- Users can view item details
- Users can edit item
- Users can delete item

3.6 Outfit Recommendations
- Users can generate outfit suggestions
- Recommendations use only owned clothes
- Users can save outfits

3.7 Stylist
- Users can describe their situation in free text (e.g. "Brunch with my in-laws, 58°F")
- Users can select one or more occasions: Work, Casual, Date, Gym, Travel, Evening
- Users can select one or more vibes: Minimal, Soft, Edgy, Bold, Cozy, Preppy
- The system generates up to 3 outfit suggestions drawn exclusively from the user's own wardrobe
- Each suggestion shows a confidence percentage, an outfit name, a reason sentence, and thumbnails of the component items
- Users can regenerate suggestions without changing their filters
- Users can react to a suggestion (thumbs up, thumbs down, bookmark)
- Vibe chips are pre-populated from the user's saved style preferences on first load

3.8 Wear History
- Users can mark items/outfits as worn
- System stores wear history

3.9 Notifications
- Daily outfit prompt notifications

3.10 Weather & Occasion
- Users can request suggestions based on:
  - weather
  - occasion

4. Non-Functional Requirements
- Performance: <2s screen load
- Security: Auth required for all data
- Scalability: Supabase backend
- Usability: Simple mobile UI


5. Assumptions
- AI tagging not implemented
- Recommendation engine not implemented
- Chat is stubbed

6. Application Structure and Screens

6.1 Navigation Structure
The application uses a bottom tab navigation system with the following five primary sections:
- Home
- Wardrobe
- Add
- Stylist
- Me

Each tab contains its own stack-based navigation for related screens.

6.2 Home Section
Purpose: Provide daily value and quick access to recommendations and actions.

Screens:
- Home Dashboard
  - Displays greeting
  - Displays weather-based prompt
  - Shows recommended outfits
  - Provides quick actions:
    - Add clothing item
    - Open stylist chat
    - View saved outfits
    - Log worn outfit

- Recommendation Results
  - Displays generated outfit suggestions
  - Allows saving outfits

6.3 Wardrobe Section
Purpose: Manage and browse the user's full wardrobe — both clothing items and outfits — via top subtabs.

Top subtabs:
- Clothing (default): grid of all clothing items with filter support
- Outfits: list of saved outfits and generated recommendations

Screens under Clothing subtab:
- Clothing Grid
  - Displays all clothing items in a grid layout
  - Supports filtering by category, color, and style

- Clothing Item Detail
  - Displays full image and metadata
  - Shows last worn date
  - Allows marking as favorite
  - Allows editing and deleting item

- Edit Clothing Item
  - Allows user to update clothing metadata

Screens under Outfits subtab:
- Saved Outfits Feed
  - Displays all saved outfits in a scrollable list

- Outfit Detail
  - Displays outfit composition
  - Allows marking as favorite
  - Allows marking as worn
  - Allows providing feedback

- Recommendation Results
  - Displays generated outfit suggestions
  - Allows saving selected outfits

6.4 Add Section
Purpose: Capture and add new clothing items to the wardrobe.

Screens:
- Add Entry
  - Options to take photo or upload from gallery

- Camera Screen
  - Allows user to capture a clothing item image

- Photo Preview
  - Displays captured image
  - Allows retake or proceed

- AI Review Screen
  - Displays AI-detected clothing attributes
  - Allows user to review and modify attributes before saving

6.5 Stylist Section
Purpose: AI-powered personal styling assistant that generates outfit suggestions from the user's own wardrobe.

Screens:
- Stylist (index)
  - Header with italic headline "What are you dressing for?"
  - Free-text prompt input with keyword-to-occasion parsing
  - Context strip showing today's date and wardrobe item count
  - Occasion chip row (Work / Casual / Date / Gym / Travel / Evening) — multi-select
  - Vibe chip row (Minimal / Soft / Edgy / Bold / Cozy / Preppy) — multi-select, pre-populated from preferences
  - "For you" section with up to 3 outfit suggestion cards
  - Regenerate button reshuffles suggestions without changing filters
  - Each outfit card shows: look number, confidence %, outfit name, reason, item thumbnails, feedback actions, "Wear today" CTA
  - Empty states for: no wardrobe items, no filters selected, no matching combos

6.6 Me Section
Purpose: Manage user account, preferences, and settings.
Status: Planned — structure and logic not yet finalized.

6.7 Onboarding Flow
- Collects user preferences:
  - favorite colors
  - disliked colors
  - preferred styles
  - sizes
- Occurs after initial signup or first login if preferences are missing

6.8 Additional Considerations
- Wear history may be surfaced in future screens or integrated into Home or Profile
- Chat functionality is included in UI but uses stubbed backend responses in v1
